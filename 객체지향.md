<h2>
추상화
</h2>
<h3>
추상화란 객체들의 공통 속성이나 기능을 묶어 이름을 붙이는 것을 말한다<br><br>
예를 들어 물고기, 사자, 토끼, 뱀이 있을때 이들은 각각의 객체이다.<br><br>
그리고 이 객체들을 "동물", "생물" 이라고 묶는 것을 추상화라고 한다.<br><br>
이때 "동물"이라는 단어는 우리가 볼 수 없는 명사이므로 추상명사라고 부른다.<br><br>
그래서 이런 단어들로 객체들을 묶는 걸 추상화라고 하는 것이다.<br><br>
객체지향에서의 추상화는 "클래스를 정의하는 것"이라고 한다.<br><br>
여기서 클래스가 바로 추상객체(추상명사)에 해당하는 것이다.<br><br>
객체지향으로 예를 하나 더 들어보자.<br><br>
홍길동 교수, 이순신 교수, 허준 교수가 있다고 하자.<br><br>
우리는 이 교수들을 객체로 보고 공통된 속성과 오퍼레이션(행위)으로 '교수'라는 클래스를 정의한다고 해보자.<br><br>
공통된 속성으로 이름, 주민등록번호, 강의 분야, 주소, 전화번호 등을 추출하고,<br><br>
공통된 행위로써 '강의하다', '채점하다', '과정 등록하다'를 추출했다고 하면,<br><br>
그를 포함한 클래스가 생성된다.<br><br>
추상화는 다른 객체들과 구분되는 핵심적인 특징들에만 집중함으로써, 복잡도를 관리할 수 있도록 한다.<br><br>
하지만 추상화는 문제 영역과 관점에 의존적이다.<br><br>
무슨 말인지 궁금한가?
예를 들어보자.
만약 "학생"이라는 클래스와 "청소년"이라는 클래스가 있다고 해보자.
"학생"에서는 "배움"에 관점을 두고
"청소년"에서는 "발달"에 관점을 둔다.
두 클래스 모두 "배움"과 "발달"이라는 속성과 오퍼레이션을 가진다.
하지만 어디에 관점을 두고 문제를 푸느냐에 따라
어떤 영역에서 중요한 것이 다른 영역에서는 그렇지 않을 수도 있다.<br><br></h3>
<h2>
캡슐화
</h2>
<h3>
캡슐화란 객체가 맡은 역할을 수행하기 위한 하나의 목적으로<br><br>
변수와 함수를 하나로 묶는것을 말한다.<br><br>
또한 데이터는 외부에서 절대 직접적으로 접근을 하면 안되고,<br><br>
함수를 통해서만 접근해야하는데 이를 가능하게 해주는 게 캡슐화다.<br><br>
그렇다면 캡슐화는 왜 객체들을 묶고 외부의 직접적인 접근을 막는걸까?<br><br>
우선, 객체들을 묶으면 프로그램을 바라보는 단위가 커진다.<br><br>
이전의 프로그래밍 언어인 C언어는 프로그램을 함수 단위로 구조화할 수 있으나,<br><br>
프로그램 소스가 커지면 이해하기 어렵고 관리가 힘들어 질 수 있었다.<br><br>
그러나 객체지향 프로그램에서는 프로그램을 클래스 단위로 바라보게 됨으로써<br><br>
좀더 복잡하고 커다란 소스코드도 쉽게 이해하게 되었다.<br><br>
왜냐하면 클래스 내부에 여러 함수를 내포할 수 있기 때문에<br><br>
프로그램 소스 코드를 바라보는 단위가 커졌으며,<br><br>
그로 인해 프로그램 관리가 좀 더 수월해진 것이다.<br><br>
두번째, 내부를 숨김으로써 내부를 좀더 자유롭게 변경할 수 있게 되었다.<br><br>
이전의 함수 중심적인 구조적 프로그래밍 언어에서는<br><br>
프로그램 내부에서 데이터가 어디서 어떻게 변경되는지 파악하기 어려웠고,<br><br>
그로 인해 유지 보수가 힘들었기 때문에<br><br>
객체 지향에서는 클래스 내부의 데이터를 외부에서 참조하지 못하도록 차단하였다.<br><br>
이렇게 차단하는 개념을 "정보 은닉"이라고 하며<br><br>
이것이 바로 "캡슐화"라는 개념이다.<br><br>
</h3>
<h2>
상속성
</h2>
<h3>
"상속"이란 상위 개념의 특징을 하위 개념이 물려받는 것을 말한다.
간단하게 자동차로 예시를 들어보자.
상위 개념은 자동차이다.
기름을 넣으면 바퀴가 굴러가는 그 자동차다.
그런데 기름을 넣고 바퀴가 굴러가는 기능 말고도
위에 뚜껑이 열리는 자동차를 만든다고 하자.
우린 방금 스포츠카라는 하위 개념을 만들었다.
그런데 스포츠카에 자동차의 특징이 없는가?
아니다. 오히려 자동차의 특징을 고스란히 가지고 있다.
이를 바로 상속이라 한다.
상속을 이용하면 하위클래스에서 속성이나 오퍼레이션을 다시 정의하지 않아 코드가 줄어든다.
그래서 좀 더 범용성있게 사용할 수 있다.
앞서 말했듯이 하위 클래스는 상위 클래스가 가지고 있는 모든 자료와 메소드를 물려받아 자유롭게 사용할 수 있다.
또한 자신만의 자료와 메소드를 추가적으로 덧붙임으로써 새로운 형태의 클래스로 발전하게 될 수도 있다.(자동차->스포츠카처럼)
</h3>
