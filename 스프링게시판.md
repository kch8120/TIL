### 1. 뼈대
#### 스프링 게시판의 뼈대인 Spring Boot, Spring MVC는 나무로 비유하면 기둥과 같다.
#### Spring Boot는 기본 설정의 복잡함을 줄여주고 내장 서버를 제공하여 즉시 실행 가능한 애플리케이션을 만든다.
#### Spring MVC에서 MVC는 Model-View-Controller를 뜻한다.
#### 즉, 사용자의 요청을 받고, 데이터를 처리하고, 화면을 보여주는 역할을 나누는 Model-View-Controller 패턴을 구현한다.
#### 여기에서 사용자의 요청을 받는 것이 Controller, 데이터를 처리하는 것이 Model, 화면에 보여주는 게 View다.

### 2. 데이터
#### 나무의 뿌리와 같은 역할을 하는 데이터베이스는 게시글을 저장하고 불러온다.
#### 데이터베이스는 주로 MySQL이나 PostgreSQL 같은 RDBMS를 사용하며, SQL을 직접 쓰기보다 자바 객체와 DB테이블을 매핑하여 데이터를 다룸으로써 생산성을 매우 높일 수 있다.
##### 🥱🥱🥱😴😴😴(이해 못함)
##### 그러니까 정리하자면
##### 1. 데이터베이스는 게시글을 저장하고 불러온다.
##### 2. 데이터베이스에는 MySQL이나 PostgreSQL을 사용한다.
##### 3. SQL 직접 사용 대신 자바 객체와 DB테이블을 매핑하여
##### 생산성을 매우 높일 수 있다.
### 3. 화면 구현
#### 화면은 나무의 이파리 같은 역할을 한다.
#### 사용자에게 보여지는 화면을 만드는 방식은
#### 서버 사이드 렌더링과 클라이언트 사이드 렌더링이 있는데,
#### 서버 사이드 렌더링은 서버에서 화면을 다 만들어서 보내주는 방식이다.
#### Thymeleaf는 Spring에서 가장 권장하는 서버 사이드 렌더링이며, HTML 태그에 Spring데이터를 자연스럽게 녹여낼 수 있다.
#### 클라이언트 사이드 렌더링은 서버가 데이터만 보내고, 화면은 브라우저가 그리는 방식이다.
#### React/Vue.js는 현대적인 웹 UI를 만들 때 주로 사용하는 클라이언트 사이드 렌더링이고, 이 경우 Spring은 REST API 서버 역할을 한다.

### 4. 도구
#### 나무의 거름과 비료 역할을 하는 도구에는 Build Tool, Lombok 등이 있다.
#### Build Tool은 라이브러리를 관리하고 빌드하는 Gradle(또는 Maven)을 말한다.
#### Lombok은 자바의 반복적인 코드를 어노테이션 하나로 줄여주어 코드를 깔끔하게 만든다.
##### 여기는 드립칠게 없네ㅠㅠ
### 5. 흐름
#### 일반적으로 게시판의 데이터는 Controller->Service->Repository->DB 순서로 흐른다.
##### Controller는 사용자의 클릭(글쓰기, 버튼 등)을 받는다.
##### Service는 '비밀번호가 맞아야 게시글 저장'같은 비즈니스 로직을 처리한다.
##### Repository는 실제 DB에 INSERT 명령을 내린다.
