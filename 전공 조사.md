<h2><b>백엔드 개발자가 뭘까?</b></h2><br><br>
<h3><b>백엔드 개발자는</b><br><br>
애플리케이션을 만들고 유지보수하는데 필요한 컴퓨터 시스템을 선택하고 설정하며,<br><br>
이를 관리하고 안정적으로 운영한다.<br><br>
<br><br>또 클라우드 서비스를 활용하여 데이터를 저장하며,<br><br>
이 모든 것을 조합해서 사용자에게 원활한 애플리케이션을 제공한다.<br><br>
또한 데이터베이스를 관리하고,<br><br>
데이터를 찾아오거나 저장하며,<br><br>
데이터가 안전하게 보관되도록 노력하는데,<br><br>
이런 작업을 통해 웹사이트나 앱이 원활하게 동작하고 필요한 정보를 제공할 수 있게 된다.<br><br>
또, API를 설계하고 구현하여 클라이언트 애플리케이션과 원활한 대화를 할 수 있게 도와준다.<br><br>
즉, 백엔드 개발자는 자바, 파이썬 등을 이용하여<br><br>
화면 상에는 보이지는 않는 서버, 데이터 저장, 로그인 처리 등의 핵심 기능을 구현한다.<br><br>
<h4><b>서버란?</b><br><br>
서버는 모든 시스템의 시작점이다.<br><br>
서버는 사용자의 요청을 받아 처리하고, 그에 대한 응답을 돌려주는 컴퓨터다.<br><br>
(실제로는 하나의 컴퓨터일 수도 있고,<br><br> 여러 대의 서버가 역할을 나눠서 처리할 수도 있다.<br><br>
이때 이를 분산 서버라고 한다.)<br><br>
<b>API란?</b><br><br>
API는 프론트엔드와 대화하는 통로다.<br><br>
API는 프론트엔드가 서버에 필요한 작업을 요청하고, <br><br>서버가 그 결과를 돌려주는 약속된 방법이다.<br><br>
대부분 REST API(전체 응답을 받는 형식) 또는 GraphQL(필요한 데이터만 선택하는 요청하는 방식) 형식으로 사용된다.<br><br>
<b>데이터베이스란?</b><br><br>
데이터베이스는 데이터를 저장하는 창고다.<br><br>
데이터베이스는 회원정보, 게시글, 주문내역 등을 안전하고 체계적으로 저장한다.<br><br>
MySQL 등의 SQL계열과 MongoDB 등의 NoSQL 계열로 나뉜다.<br><br>
</h4>
<h3><br><br>
근데 백엔드 개발자가 있으면 백스타트 개발자도 있나요? 깔깔<br><br>
아니요..;; 그 대신 프론트엔드개발자라는 직종이 있습니다.<br><br>
<hr></h3>
<h2><b>프론트엔드 개발자가 뭘까?</b></h2>
<h3><b>프론트엔드 개발자는</b><br><br>
웹사이트나 앱에서 사용자의 눈에 보이는 버튼, 메뉴 등을 배치함으로써
사용자와 상호작용을 하는 UI를 만든다.<br><br>
하지만 프론트엔드 개발자는 이러한 UI를 구현하는 것에 그치지 않고,<br><br>
엔지니어로서 요구사항을 분석하고, 애플리케이션이 지속적으로 변경 가능하도록<br><br>
유연한 설계를 고민하고, 프로젝트 상황에 맞는 기술을 개바라고,<br><br>
테스트 코드를 작성하여 제품의 품질을 유지하고, <br><br>
안전하게 제품을 전달할 수 있도록 배포 작업을 수행하는 등<br><br>
소프트웨어 엔지니어링 업무를 수행한다.<br><br>
</h3>
<h4><b>UI란?</b><br><br>
UI는 User Interface의 약자로, 애플리케이션에서 사용자가 눈으로 보고 상호작용하는 모든 요소이다.<br><br>
즉, 우리가 특정 서비스를 사용할 때 특정 기능이 위치하고 있는 곳이다.<br><br>
</h4>
<hr>
<h2><b>클라우드 개발자가 뭘까?</b><br><br></h2>
<h3><b>클라우드 개발자는</b><br><br>
클라우드 환경에서 동작하는<br><br>
소프트웨어와 애플리케이션을 설계하고 개발하는 IT전문가다.<br><br>
이들은 단순히 코딩만 하는 게 아니다.<br><br>
AWS, Azure, Google Cloud같은 클라우드 플랫폼을 활용해 인프라를 설계하고,<br><br>
서비스가 안정적으로 돌아가도록 관리한다.<br><br>
때로는 백엔드 개발자의 역할도 겸하면서,<br><br>
데이터 관리와 보안 설계까지 챙기는 전문가라고 보면 된다.
클라우드 개발자는 AWS나 Azure같은 플랫폼에서 서버와 네트워크를 설계하고 구축하고,<br><br>
클라우드 환경에서 서비스가 매끄럽게 돌아가도록 설계하고 유지보수한다.<br><br>
또, DevOps 기술을 활용해 배포 과정을 자동화하고, 비용을 효율화하고,<br><br>
데이터를 안전하게 보호하고, 클라우드 시스템의 보안을 강화한다.<br><br>
<h4>DevOps란?
Development와 Operation의 합성어로,<br><br>
개발 담당자와 운영 담당자가 연계하여 협력하는 개발 방법론이다.<br><br>
한마디로 개발부서와 운영부서 간의 원활한 소통을 기반으로 한다.<br><br>
데브옵스 기술을 사용하면 작업 속도가 빨라지고,<br><br>
개발자와 운영 부서간의 협력을 통해 효과적인 팀 구축이 가능하다.<br><br></h4>
<h3>근데 클라우드 개발자가 구름을 개발하는게 아니었네요? 신인 줄 알았어요 ㄷㄷ<br><br>
ㅎㅎ...;; 이제 클라우드 개발자가 뭔 지 알겠죠?</h3>
<hr>
<h2><b>UI/UX 디자이너가 뭘까?</b></h2>
<h3><b>UI/UX 디자이너는</b>
웹사이트나 모바일 앱을 사용자가 쉽고 편리하게 사용할 수 있도록 설계하는 전문가다.<br><br>
UX 디자인은 사용자가 제품을 왜, 어떻게, 무엇을 사용하는 지를 분석하고 설계한다.<br><br>
단순히 예쁜 디자인을 넘어, 문제 해결과 사용성 개선에 집중한다.<br><br>
UX디자이너는 사용자의 니즈를 파악하고, 대표 사용자 유형을 설정하고,<br><br>
서비스 이용 흐름을 시각화하고, 콘텐츠 배치, 메뉴 구성을 한다.<br><br>
또, 프로토타입 제작 및 사용성 테스트 및 개선도 한다.<br><br>
목표는 사용자가 이해하기 쉽고, 원하는 작업을 빠르게 완료할 수 있도록 돕는 것이다.<br><br>
UI 디자인은 화면에서 사용자가 실제로 마주하는 요소들을 시각적으로 설계한다.<br><br>
색상, 버튼, 폰트, 애니메이션 등 보이는 모든 디자인이 포함된다.
UI디자이너는 컬러, 타이포그래피, 아이콘 등을 디자인하고,<br><br>
버튼, 입력창, 탭 등 시각적 인터페이스를 설계하고,
사용자 행동에 따른 반응을 설계하고,
일관성 있는 디자인을 위한 가이드를 구축한다.<br><br>
목표는 예쁘고 보기 좋은 디자인이 아니라, 직관적이고 일관된 사용 경험이다.<br><br>
</h3>
<h4>UX란?
User Experience의 약자로, 고객이 특정 서비스를 사용하면서 느낀 경험이다.<br><br>
즉, UX는 고객이 느낀 감정이다.<br><br>
개발 담당자와 운영 담당자가 연계하여 협력하는 개발 방법론이다.<br><br>
한마디로 개발부서와 운영부서 간의 원활한 소통을 기반으로 한다.<br><br>
데브옵스 기술을 사용하면 작업 속도가 빨라지고,<br><br>
개발자와 운영 부서간의 협력을 통해 효과적인 팀 구축이 가능하다.<br><br><br><br></h4>

<h2><b>그렇다면 프론트엔드와 백엔드는 어떻게 상호작용할까?</b></h2>
<h3>프론트엔드와 백엔드는 마치 서로 편지를 주고받는 친구들과 같다.<br><br>
프론트엔드는 우리가 보는 앱 화면이고,<br><br>
백엔드는 그  뒤에서 정보를 처리하는 머리역할이다.<br><br>
프론트엔드가 보내는 편지에는 "이 데이터를 줘"라는 요청이 들어있다.<br><br>
이 편지를 보내기 위해서 우리는 코드를 작성해야 한다.<br><br>
백엔드는 편지를 받고 그 요청에 맞게 데이터를 준비해서 보내는 역할을 한다.<br><br>
응답을 받은 프론트엔드는 데이터를 화면에 표시하고,<br><br>
사용자는 그 정보를 앱에서 확인할 수 있게 된다.
즉, 프론트엔드는 데이터를 요청하고 화면에 표시하는 역할을 하고,<br><br>
백엔드는 그 요청에 응답하고 데이터를 제공하는 역할을 한다.<br><br>
