<h2>HTTP가 뭘까?<br><br></h2>
<h3>
HTTP는 <mark>HyperText Transfer Protocol</mark> 약자이다.<br><br>
뜻은 하이퍼텍스트 전송 규약이다.<br><br>
그런데 우리가 못봤던 단어가 있다.<br><br>
하이퍼텍스트가 뭘까?<br><br>
하이퍼텍스트는 하이퍼링크와 비슷한 개념이다.<br><br>
단지 하이퍼링크를 좀 더 고급지게 개조시킨 것이 하이퍼텍스트인 것이다.<br><br>
HTTP는 기본적으로 <mark>클라이언트-서버</mark> 모델을 사용하는 요청-응답 기반 프로토콜이다.<br><br>
클라이언트가 뭔가를 요청하면, 서버가 응답하는 과정에서 사용되는 프로토콜이 HTTP인 것이다.<br><br><br>
예를 들어 웹사이트에 방문해서 영상을 보거나, 스마트폰으로 앱을 사용하거나,<br><br>
브라우저로 데이터를 업로드할 때 클라이언트는 서버에 HTTP 요청 메시지를 보낸다.<br><br><br>
그럼 HTML 파일, 이미지, 동영상 같은 콘텐츠 리소스를 제공하는 서버는<br><br>
클라이언트에 응답 메시지를 회신한다.<br><br></h3>
<h2>
HTTP Method가 뭘까?<br><br></h2>
<h3>
HTTP가 하는 응답에는 그 안에 포함된 데이터를 설명하는 HTTP 헤더가 추가로 달려 있다.<br><br>
이 헤더는 유저에게는 보이지 않지만, 브라우저와 서버는 바로 이 HTTP 헤더로 정보를 읽는다.<br><br>
브라우저와 서버는 헤더와 함께 요청한 결과를 표시하는 HTTP 상태 코드를 전달하기도 하는데,
이는 다음 챕터인 Status Code에서 알아보도록 하자.<br><br></h3>
<h2>HTTP Status Code가 뭘까?</h2>
<h3>1xx = 정보 상태 코드<br><br>
100 = 계속(요청의 초기 부분이 수신되었고, 서버에서 아직 거부되지 않았음 서버는 요청을 완전히 수신하고 처리한 후 최종 응답을 보낼 에정.)<br><br>
101 = 스위칭 프로토콜(서버가 클라이언트가 보낸 '이 연결에서 사용되는 애플리케이션 프로토콜 변경 요청'을 이해하였고 수락할 의사가 있음.)<br><br>
2xx = 성공 등급의 상태 코드<br><br>
200 = OK(요청이 성공했음.)<br><br>
201 = 생성됨(요청이 완료되어 하나 이상의 새 리소스가 생성되었음.)<br><br>
202 = 승인됨(요청이 처리 대상으로 접수되었지만 처리가 아직 완료되지 않았음)<br><br>
203 = 비공식 정보(요청은 성공했지만 원본 서버(200번) 응답 내용과 다르게 포함된 콘텐츠가 수정되었음.)<br><br>
204 = 내용 없음(서버가 요청을 성공적으로 처리했으며 응답 콘텐츠에 추가로 전송할 콘텐츠가 없음.)<br><br>
205 = 콘텐츠 재설정(서버가 요청을 처리했으며, 사용자 에이전트가 해당 요청을 발생시킨 "문서 보기"를 원래 서버에서 수신한 상태로 재설정 하기를 원하다는 것을 나타냄(ex. 사용자가 데이터 입력을 지원하는 콘텐츠를 받고 해당 공간에 데이터를 입력한 후 입력한 데이터를 제출하면 입력한 데이터는 남아있지 않고 초기화됨.))<br><br>
206 = 부분 내용(서버가 선택된 표현의 하나 이상의 부분을 전송하여 대상 리소승 대한 범위 요청을 성공적으로 이행하고 있음.)<br><br>
3xx = 리디렉션 상태 코드<br><br><br><br>
300 = 다중선택(대상 리소스에 여러 표현이 있으며 각 표현에는 더 구체적인  식별자가 있고 대안에 대한 정보가 제공되어 사용자가 요청을 이러한 식별자ㅜㅇ 하나 이상으로 리디렉션하여 선호하는 표현을 선태갈 수 있음.)<br><br>
301 = 영구 이동됨(대상 리소스에 새 영구 URI가 할당되었으며 향후 이 리소스를 참조할 때는 포함된 URI 중 하나를 사용해야 함.)<br><br>
302 = 발견됨(대상 리소스가 일시적으로 다른 URI에 있음.)<br><br>
303 = 기타 참조(서버가 사용자 에이전트를 Location 헤더 필드의 URI로 지정된 다른 리소스로 리디렉션하고있음.)<br><br>
304 = 수정되지 않음(조건부 요청이 수신되었으며 조건이 충족되지 않았다면 200 응답이 반환되었을 것임을 나타냄.)<br><br>
4xx = 클라이언트 오류<br><br>
400 = 잘못된 요청(서버가 클라이언트 오류로 인해 요청을 처리할 수 없거나 처리하지 않을 것임을 나타냄.)<br><br>
401 = 무단 접근(대상 리소스에 대한 유효한 인증 자격 증명이 부족하여 요청이 적용되지 않았음.)<br><br>
403 = 금지됨(서버가 요청을 이해했지만 처리를 거부했음.)<br><br>
<span style="color:red">404</span> = 찾을 수 없음(가장 대표적인 코드, 서버가 원본 서버가 대상 리소스에 대한 최신 표현을 찾지 못했거나 존재한다는 사실을 알리고 싶지 않음.)<br><br>
405 = 메서드가 허용되지 않음(요청 라인에 포함된 메서드가 원본 서버에는 알려져 있지만 대상 리소스에서는 지원되지 않음.)<br><br>
406 = 허용되지 않음(대상 리소스의 요청에서 수신된 섹션에 따라 사용자 에이전트가 허용할 수 있는 현재 표현이 없으며 서버가 기본 표현을 제공할 의사가 없음.)<br><br>
407 = 프록시 인증 필요(401(권한 없음)과 유사하지만, 클라이언트가 이 요청에 대해 프록시를 사용하기 위해 자체 인증이 필요함을 나타냄.)<br><br>
408 = 요청 시간 초과(서버가 대기할 준비가 된 시간 내에 완전한 요청 메싲를 수신하지 못했음.)<br><br>
409 = 갈등(대상 리소스의 현재 상태와의 충돌로 인해 요청을 완료할 수 없음을 나타냄.)<br><br>
410 = 사라짐(원본 서버에서 대상 리소스에 대한 액세스가 더 이상 불가능하며 이러한 상태가 영구적일 가능성이 높다는 것을 나타냄.)<br><br>
411 = 필요한 길이(서버가 요청 콘텐츠의 길이가 정의되지 않은 요청을 수락하지 않음.)<br><br>
412 = 사전 조건 실패(요청 헤더 필드에 지정된 하나 이상의 조건이 서버에서 테스트 했을 때 거짓으로 평가되었음을 나타냄.)<br><br>
413 = 콘텐츠 용량이 너무 큽니다(서버가 요청내용을 처리할 수 있을 용량보다 커서 처리하지 못하고 있음을 나타냄.)<br><br>
414 = URI 너무 깁니다(서버가 대상 URI가 서버에서 해석할 수 있는 길이보다 길어 요청을 처리할 수 없음을 나타냄.)<br><br>
415 = 지원되지 않는 미디어 유형(원본 서버가 대상 리소스에서 이 메서드가 지원하지 않는 형식의 콘텐츠이기 때문에 요청 처리를 거부하고 있음을 나타냄.)<br><br>
416 = 버위가 만족되지 않음(요청이 Range 헤더 필드에 있는 범위 집합이 거부되었음을 나타냄.)<br><br>
417 = 기대 실패(요청의 헤더 필드에 명시된 예상을 수신 서버중 하나 이상에서 충족할 수 없었음을 나타냄.)<br><br>
421 = 잘못된 요청(요청이 대상 URI에 대한 권한 있는 응답을 생성할 수 없거나 생성할 의사가 없는 서버로 전송되었음을 나타냄.)<br><br>
422 = 처리할 수 없는 콘텐츠(서버가 요청 콘텐츠의 콘텐츠 유형을 이해하고 요청 콘텐츠의 구문이 올바르지만 포함된 명령어를 처리할 수 없음을 나타냄.)<br><br>
426 = 업그레이드 필요(서버가 현재 프로토콜을 사용하여 요청을 수행하는 것을 거부하지만 클라이언트가 다른 프로토콜로 업그레이드 한 후에는 수행할 의향이 있음을 나타냄.)<br><br>
5xx = 서버 오류<br><br><br>
500 = 내부 서버 오류(서버가 요청을 처리할 수 없게 만드는 예기치 않은 상황을 만났음을 나타냄.)<br><br>
501 = 구현되지 않음(서버가 요청을 처리하는 데 필요한 기능을 지원하지 않음을 나타냄.)<br><br>
502 = Bad Gateway(서버가 게이트웨이 또는 프록시 역할을 하는 동안 요청을 처리하려고 시도하는 과정에서 액세스한 서버로부터 유효하지 않은 응답을 수신했음을 나타냄.)<br><br>
503 = 서비스 이용 불가(서버가 일시적인 과부하 또는 예정된 유지 보수로 인해 현재 요청을 처리할 수 없음을 나타냄.)<br><br>
504 = 게이트웨이 시간 초과(서버가 게이트웨이 또는 프록시 역할을 하는 동안 요청을 완료하기 위해 액세스해야 하는 상위 서버로부터 적시에 응답을 받지 못했음을 나타냅니다.<br><br>
505 = HTTP 버전이 지원되지 않습니다(서버가 요청 메시지에 사용된 HTTP 주 버전을 지원하지 않거나 지원을 거부함을 나타냄.)<br><br></h3>
<h2>HTTP의 무상태성이 뭘까?</h2><br><br>
<h3>무상태 프로토콜이란 서버가 클라이언트의 상태를 보존하지 않는 특징이다.<br><br>
서버가 확장성에 용이하다는 장점이 있지만, 클라이언트가 추가 데이터를 전송해야 하는 단점이 있다.<br><br>

```
상태 유지(Stateful)란?
클라이언트의 요청1에 대한 상태를 해당 요청을 받은 서버A가 기억하고 있는 것으로
항상 같은 서버A가 유지되어야 할 것이다. 만약 서버A에서 장애가 난다면
유지되던 상태 정보가 없어져 버림로 처음부터 다시 요청을 해야 한다.
```
무상태 프로토콜이라면 클라이언트 요청1을 보낼 때, 데이터를 다 담아서 보내기 때문에 아무 서버나 호출할 수 있다.<br><br>
무상태는 응답 서버를 쉽게 바꿀 수 있기 때문에 무한한 서버 증설이 가능하다.<br><br>
```
무상태(stateless)의 한계
로그인이 필요 없는 단순한 서비스 소개 화면 같은 경우엔 무상태로 설계할 수 있지만,
로그인이 필요한 서비스라면 유저의 상태를 유지해야 되기 때문에
브라우저 쿠키, 서버 세션, 토큰 등을 이용해 상태를 유지한다.
이러한 상태 유지는 최소한만 사용한다.
```

<h2>감사합니다</h2>
