<h2>HTTP가 뭘까?<br><br></h2>
<h3>
HTTP는 <mark>HyperText Transfer Protocol</mark> 약자이다.<br><br>
뜻은 하이퍼텍스트 전송 규약이다.<br><br>
그런데 우리가 못봤던 단어가 있다.<br><br>
하이퍼텍스트가 뭘까?<br><br>
하이퍼텍스트는 하이퍼링크와 비슷한 개념이다.<br><br>
단지 하이퍼링크를 좀 더 고급지게 개조시킨 것이 하이퍼텍스트인 것이다.<br><br>
HTTP는 기본적으로 <mark>클라이언트-서버</mark> 모델을 사용하는 요청-응답 기반 프로토콜이다.<br><br>
클라이언트가 뭔가를 요청하면, 서버가 응답하는 과정에서 사용되는 프로토콜이 HTTP인 것이다.<br><br><br>
예를 들어 웹사이트에 방문해서 영상을 보거나, 스마트폰으로 앱을 사용하거나,<br><br>
브라우저로 데이터를 업로드할 때 클라이언트는 서버에 HTTP 요청 메시지를 보낸다.<br><br><br>
그럼 HTML 파일, 이미지, 동영상 같은 콘텐츠 리소스를 제공하는 서버는<br><br>
클라이언트에 응답 메시지를 회신한다.<br><br></h3>
<h2>
HTTP Method가 뭘까?<br><br></h2>
<h3>
HTTP가 하는 응답에는 그 안에 포함된 데이터를 설명하는 HTTP 헤더가 추가로 달려 있다.<br><br>
이 헤더는 유저에게는 보이지 않지만, 브라우저와 서버는 바로 이 HTTP 헤더로 정보를 읽는다.<br><br>
브라우저와 서버는 헤더와 함께 요청한 결과를 표시하는 HTTP 상태 코드를 전달하기도 하는데,<br><br>
이 상태 코드의 예시는 다음과 같다.<br><br>
<mark>
1xx = 정보 상태 코드
100 = 계속(요청의 초기 부분이 수신되었고, 서버에서 아직 거부되지 않았음 서버는 요청을 완전히 수신하고 처리한 후 최종 응답을 보낼 에정.)
101 = 스위칭 프로토콜(서버가 클라이언트가 보낸 '이 연결에서 사용되는 애플리케이션 프로토콜 변경 요청'을 이해하였고 수락할 의사가 있음.)<br><br>
2xx = 성공 등급의 상태 코드
200 = OK(요청이 성공했음.)
201 = 생성됨(요청이 완료되어 하나 이상의 새 리소스가 생성되었음.)
202 = 승인됨(요청이 처리 대상으로 접수되었지만 처리가 아직 완료되지 않았음)
203 = 비공식 정보(요청은 성공했지만 원본 서버(200번) 응답 내용과 다르게 포함된 콘텐츠가 수정되었음.)
204 = 내용 없음(서버가 요청을 성공적으로 처리했으며 응답 콘텐츠에 추가로 전송할 콘텐츠가 없음.)
205 = 콘텐츠 재설정(서버가 요청을 처리했으며, 사용자 에이전트가 해당 요청을 발생시킨 "문서 보기"를 원래 서버에서 수신한 상태로 재설정 하기를 원하다는 것을 나타냄(ex. 사용자가 데이터 입력을 지원하는 콘텐츠를 받고 해당 공간에 데이터를 입력한 후 입력한 데이터를 제출하면 입력한 데이터는 남아있지 않고 초기화됨.))
3xx=영구 이동(리디렉션)<br><br>
4xx=찾을 수 없음<br><br>
4xx=법적인 이유로 사용 불가<br><br>
5xx=내부 서버 오류<br><br></mark>
여기서 가장 대표적인 코드는 바로 404 코드로<br><br>
인터넷을 사용할 때 <ins>404 not found</ins> 모습으로 흔히 나타나며<br><br>
404 오류라고도 부른다..<br><br></h3>
