<h2>HTTP가 뭘까?<br><br></h2>
<h3>
HTTP는 <mark>HyperText Transfer Protocol</mark> 약자이다.<br><br>
뜻은 하이퍼텍스트 전송 규약이다.<br><br>
그런데 우리가 못봤던 단어가 있다.<br><br>
하이퍼텍스트가 뭘까?<br><br>
하이퍼텍스트는 하이퍼링크와 비슷한 개념이다.<br><br>
단지 하이퍼링크를 좀 더 고급지게 개조시킨 것이 하이퍼텍스트인 것이다.<br><br>
HTTP는 기본적으로 <mark>클라이언트-서버</mark> 모델을 사용하는 요청-응답 기반 프로토콜이다.<br><br>
클라이언트가 뭔가를 요청하면, 서버가 응답하는 과정에서 사용되는 프로토콜이 HTTP인 것이다.<br><br><br>
예를 들어 웹사이트에 방문해서 영상을 보거나, 스마트폰으로 앱을 사용하거나,<br><br>
브라우저로 데이터를 업로드할 때 클라이언트는 서버에 HTTP 요청 메시지를 보낸다.<br><br><br>
그럼 HTML 파일, 이미지, 동영상 같은 콘텐츠 리소스를 제공하는 서버는<br><br>
클라이언트에 응답 메시지를 회신한다.<br><br></h3>
<h2>
HTTP Method가 뭘까?<br><br></h2>
<h3>
HTTP가 하는 응답에는 그 안에 포함된 데이터를 설명하는 HTTP 헤더가 추가로 달려 있다.<br><br>
이 헤더는 유저에게는 보이지 않지만, 브라우저와 서버는 바로 이 HTTP 헤더로 정보를 읽는다.<br><br>
브라우저와 서버는 헤더와 함께 요청한 결과를 표시하는 HTTP 상태 코드를 전달하기도 하는데,<br><br>
이 상태 코드의 예시는 다음과 같다.<br><br>
<mark>
1xx = 정보 상태 코드<br><br>
100 = 계속(요청의 초기 부분이 수신되었고, 서버에서 아직 거부되지 않았음 서버는 요청을 완전히 수신하고 처리한 후 최종 응답을 보낼 에정.)<br><br>
101 = 스위칭 프로토콜(서버가 클라이언트가 보낸 '이 연결에서 사용되는 애플리케이션 프로토콜 변경 요청'을 이해하였고 수락할 의사가 있음.)<br><br>
2xx = 성공 등급의 상태 코드<br><br>
200 = OK(요청이 성공했음.)<br><br>
201 = 생성됨(요청이 완료되어 하나 이상의 새 리소스가 생성되었음.)<br><br>
202 = 승인됨(요청이 처리 대상으로 접수되었지만 처리가 아직 완료되지 않았음)<br><br>
203 = 비공식 정보(요청은 성공했지만 원본 서버(200번) 응답 내용과 다르게 포함된 콘텐츠가 수정되었음.)<br><br>
204 = 내용 없음(서버가 요청을 성공적으로 처리했으며 응답 콘텐츠에 추가로 전송할 콘텐츠가 없음.)<br><br>
205 = 콘텐츠 재설정(서버가 요청을 처리했으며, 사용자 에이전트가 해당 요청을 발생시킨 "문서 보기"를 원래 서버에서 수신한 상태로 재설정 하기를 원하다는 것을 나타냄(ex. 사용자가 데이터 입력을 지원하는 콘텐츠를 받고 해당 공간에 데이터를 입력한 후 입력한 데이터를 제출하면 입력한 데이터는 남아있지 않고 초기화됨.))<br><br>
206 = 부분 내용(서버가 선택된 표현의 하나 이상의 부분을 전송하여 대상 리소승 대한 범위 요청을 성공적으로 이행하고 있음.)
3xx = 리디렉션 상태 코드<br><br>
300 = 다중선택(대상 리소스에 여러 표현이 있으며 각 표현에는 더 구체적인  식별자가 있고 대안에 대한 정보가 제공되어 사용자가 요청을 이러한 식별자ㅜㅇ 하나 이상으로 리디렉션하여 선호하는 표현을 선태갈 수 있음.)
301 = 영구 이동됨(대상 리소스에 새 영구 URI가 할당되었으며 향후 이 리소스를 참조할 때는 포함된 URI 중 하나를 사용해야 함.)
302 = 발견됨(대상 리소스가 일시적으로 다른 URI에 있음.)
303 = 기타 참조(서버가 사용자 에이전트를 Location 헤더 필드의 URI로 지정된 다른 리소스로 리디렉션하고있음.)
304 = 수정되지 않음(조건부 요청이 수신되었으며 조건이 충족되지 않았다면 200 응답이 반환되었을 것임을 나타냄.)
4xx = 클라이언트 오류<br><br>
400 = 잘못된 요청(서버가 클라이언트 오류로 인해 요청을 처리할 수 없거나 처리하지 않을 것임을 나타냄.)
401 = 무단 접근(대상 리소스에 대한 유효한 인증 자격 증명이 부족하여 요청이 적용되지 않았음.)
403 = 금지됨(서버가 요청을 이해했지만 처리를 거부했음.)
<span style="color:red">404</span> = 찾을 수 없음(가장 대표적인 코드, 서버가 원본 서버가 대상 리소스에 대한 최신 표현을 찾지 못했거나 존재한다는 사실을 알리고 싶지 않음.)
405 = 메서드가 허용되지 않음(요청 라인에 포함된 메서드가 원본 서버에는 알려져 있지만 대상 리소스에서는 지원되지 않음.)
406 = 허용되지 않음(대상 리소스의 요청에서 수신된 섹션에 따라 사용자 에이전트가 허용할 수 있는 현재 표현이 없으며 서버가 기본 표현을 제공할 의사가 없음.)
407 = 프록시 인증 필요(401(권한 없음)과 유사하지만, 클라이언트가 이 요청에 대해 프록시를 사용하기 위해 자체 인증이 필요함을 나타냄.)
408 = 요청 시간 초과(서버가 대기할 준비가 된 시간 내에 완전한 요청 메싲를 수신하지 못했음.)
409 = 갈등(대상 리소스의 현재 상태와의 충돌로 인해 요청을 완료할 수 없음을 나타냄.)
410 = 사라짐(원본 서버에서 대상 리소스에 대한 액세스가 더 이상 불가능하며
5xx=내부 서버 오류<br><br></mark>
여기서 가장 대표적인 코드는 바로 404 코드로<br><br>
인터넷을 사용할 때 <ins>404 not found</ins> 모습으로 흔히 나타나며<br><br>
404 오류라고도 부른다.<br><br></h3>
